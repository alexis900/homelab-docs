# INC — Acceso a Home Assistant fallido por certificado inválido

## Resumen

Se detectó imposibilidad de acceso a Home Assistant de manera externa debido a un problema con el certificado TLS (Este está caducado). El servicio respondía, pero el navegador rechazaba la conexión por certificado inválido/expirado. Incidente resuelto tras regenerar el certificado.

## Severidad e Impacto

- Severidad: Baja  
- Impacto: acceso web a Home Assistant bloqueado; afectaba únicamente a Internet fuera de la red local.

## Línea temporal

- **00:00** — Detección (error de certificado en acceso HTTPS).
- **00:00** — Regeneración del certificado en NPM.
- **00:00** — Acceso restaurado correctamente.

## Causa raíz

Certificado TLS inválido debido a la expiración del mismo, provocando rechazo de conexión por parte del cliente.

## Acciones realizadas

1. Acceso al firewall para aprobar todo el trafico en la VLAN20.
2. Reinicio del CT de NPM para la correcta navegación a Internet.
3. Regeneración del certificado en NPM.
4. Acceso al firewall para volver a denegar todo el tafico en la VLAN20.
5. Reinicio del CT para aplicar cambios.
6. Aplicación y validación del nuevo certificado.

## Verificación

- [x] Acceso HTTPS a NPM operativo sin errores de certificado.
- [x] Navegador ya no muestra advertencias de seguridad.

## Lecciones aprendidas

- Monitorizar expiración de certificados.
- Validar estado de certificados tras cambios en red o configuración.
