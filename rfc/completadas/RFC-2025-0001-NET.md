# Título:

Migración de Apollo a VLAN 99

# Tipo de cambio:

NET

# Fecha propuesta:

2025-07-16

# Estado:

Completado

# Criticidad:

Alta

# Justificación de criticidad:

El cambio afecta a la red de gestión y puede dejar el nodo principal
(Apollo) inaccesible si falla.

# Resumen:

Migrar la interfaz de gestión del nodo Proxmox "Apollo" desde la VLAN
por defecto a la VLAN de gestión (VLAN 99), para reforzar el aislamiento
y el control de acceso.

# Motivación:

\- Centralizar el acceso de administración.\
- Preparar la infraestructura para el clúster.\
- Cumplir con la segmentación lógica planeada.

# Dispositivo(s) afectados:

\- Nodo Apollo (Proxmox)\
- Switch SG2008\
- Router OPNsense

# Plan de acción:

1\. Crear interfaz VLAN 99 en OPNsense.\
2. Asignar IP 10.0.99.101/24 a Apollo.\
3. Configurar puerto trunk en switch.\
4. Cambiar red de administración de Proxmox.\
5. Verificar conectividad y acceso vía Web UI y SSH.

# Riesgos:

\- Pérdida temporal de acceso a Apollo.\
- Fallo en reconfiguración de red que requiera acceso físico.

# Mitigación:

\- Monitorización directa vía pantalla y teclado.\
- Backup previo de configuración de red en Proxmox.

# Resultado:

Se hace el cambio satisfactoriamente a la VLAN99 y se desactiva interfaz
de VLAN1.
