# Título:

Instalación de Uptime Kuma para monitorización de servicios

# Tipo de cambio:

MON

# Fecha propuesta:

2025-07-16

# Estado:

Completado

# Criticidad:

Media

# Justificación de criticidad:

Uptime Kuma es un sistema de monitorización no crítico pero útil para la
visibilidad del estado de servicios.

# Resumen:

Se propone instalar Uptime Kuma en una VM o CT para monitorizar
servicios internos (Home Assistant, Pi-hole, etc.).

# Motivación:

\- Detectar caídas de servicios clave de forma visual y vía alertas.\
- Mejorar el mantenimiento preventivo.\
- Registrar tiempos de disponibilidad.

# Dispositivo(s) afectados:

\- VM o CT dedicado para Uptime Kuma\
- Servicios monitorizados (solo observación, sin cambios en ellos)

\- OPNsense (reglas de firewall)

# Plan de acción:

1\. Crear CT o VM con Debian base.\
2. Instalar Node.js, PM2 y dependencias.\
3. Desplegar Uptime Kuma y acceder a la interfaz web.\
4. Configurar monitoreo de servicios internos.\
5. Probar alertas y accesos.

# Riesgos:

\- Exposición web del panel si no se protege correctamente.\
- Uso de recursos si se configura con muchas sondas.

# Mitigación:

\- Limitar el acceso por VLAN o IP.\
- Configurar alertas razonables.\
- Mantener backups periódicos del contenedor o VM.
