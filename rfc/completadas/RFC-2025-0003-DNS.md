# Título:

Actualización de IP de los servidores DNS internos

# Tipo de cambio:

NET

# Fecha propuesta:

2025-07-16

# Estado:

Completada

# Criticidad:

Media

# Justificación de criticidad:

El cambio afectó a los servidores DNS principales, con impacto potencial
en la resolución de nombres para toda la red local.

# Resumen:

Se actualizaron las IPs de los servidores DNS internos, moviéndose de la
VLAN de usuarios a la VLAN de infraestructura.

# Motivación:

\- Alinear la infraestructura de red con la nueva segmentación por
VLANs.\
- Mejorar el aislamiento y la gestión de servicios de red.

# Dispositivo(s) afectados:

\- Pi-hole\
- OPNsense\
- Clientes que usaban DNS asignado por DHCP

# Plan de acción:

1\. Modificar las IPs de Pi-hole a 10.0.30.10 y 10.0.30.11.\
2. Actualizar configuración de DNS en OPNsense y servidores DHCP.\
3. Propagar cambios en VLANs afectadas.\
4. Verificar resolución de nombres desde múltiples segmentos de red.

# Riesgos:

\- Pérdida de conectividad DNS si no se propagan bien los cambios.\
- Clientes con DNS estático apuntando a IPs antiguas.

# Mitigación:

\- Propagación ordenada en todas las VLANs.\
- Comprobación posterior desde múltiples dispositivos y segmentos.

# Resultado:

Cambio aplicado correctamente el 2025-07-16.\
Verificada resolución DNS desde VLANs 1, 30, 40 y 99. No se detectaron
incidencias.
