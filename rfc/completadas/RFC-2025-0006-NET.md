# RFC-ID:

RFC-2025-0006-NET

# Título:

Creación de la VLAN 20- DMZ para servicios expuestos

# Tipo de cambio:

NET

# Fecha propuesta:

2025-07-30

# Estado:

Completado

# Criticidad:

Media

# Justificación de criticidad:

Segmentar servicios accesibles desde Internet en una VLAN dedicada (DMZ)
mejora la seguridad al aislar estos servicios del resto de la red
interna. Esta VLAN servirá como contenedor para servicios como Nginx
Proxy Manager, y cualquier otro servicio que deba exponerse de forma
controlada hacia el exterior.

# Resumen:

Se crea una nueva VLAN en la infraestructura de red con las siguientes
características:

-   **Nombre:** DMZ

-   **VLAN ID:** 20

-   **Dirección IP de gateway:** 10.0.20.1/24

-   **Reglas iniciales del cortafuegos:**

    -   **DMZ → WAN:** Permitir solo puertos 443

    -   **DMZ → LAN:** Bloqueado por defecto

    -   **LAN → DMZ:** Permitir solo desde IPs administradoras y puertos
        específicos (ej. gestión NPM)\
        **DMZ → DNS interno:** Permitido

# Motivación:

Implementar una red segmentada y controlada para alojar servicios que
necesitan estar expuestos a Internet, reduciendo el riesgo de
comprometer la red interna en caso de incidentes de seguridad.

# Dispositivo(s) afectados:

-   **Switches afectados:** SG2008 (Trunk y VLAN Access según topología)

-   **Router:** OPNsense (interface VLAN 20 sobre interfaz física
    correspondiente)

# Plan de acción:

-   Crear la VLAN 20 en OPNsense

-   Configurar la interfaz 10.0.20.1/24

-   Aplicar reglas iniciales de firewall

-   Agregar VLAN 20 a los trunks del switch SG2008

-   Configurar acceso en la VM de NPM (bridge en Proxmox)

-   Verificar conectividad interna y externa

-   Testear reglas DNS, HTTP y HTTPS

# Riesgos:

-   Interrupción temporal del servicio NPM durante la migración.\
    Posibles errores de configuración de firewall que impida el acceso
    desde/hacia la DMZ.

-   Configuraciones incorrectas de VLAN en el switch que impidan la
    conectividad.

-   Si no se limita correctamente el tráfico desde la DMZ hacia la LAN,
    podría abrirse una puerta a vectores de ataque internos.

-   Dependencia de NAT/port forwarding en OPNsense: una mala
    configuración podría dejar el servicio inaccesible.

# Mitigación:

-   Eliminar VLAN 20 de OPNsense y switches

-   Reubicar el servicio NPM a la VLAN 30 (servidores)

-   Borrar reglas de cortafuegos relacionadas

-   Reiniciar red para aplicar cambios

# Resultado:

Creación de la VLAN 20 - DMZ para servicios expuestos completada
correctamente. La configuración y segmentación de la VLAN se realizó sin
incidencias y los servicios quedaron accesibles según lo esperado.
