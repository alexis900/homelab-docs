# RFC — Instalación de Nginx Proxy Manager (NPM) en VLAN 20

## Resumen

Se desplegó **Nginx Proxy Manager (NPM)** en un contenedor LXC dentro de la **VLAN 20 (DMZ)** para gestionar accesos externos e internos mediante proxy inverso con certificados TLS.

---

## Motivación

- Centralizar la gestión de proxy inverso.  
- Simplificar configuraciones SSL, redirecciones y accesos seguros a servicios internos.

---

## Justificación de criticidad

NPM gestiona el proxy inverso para servicios críticos; fallos en la instalación o configuración podrían afectar accesos y disponibilidad. Por ello, la criticidad es **media**.

---

## Dispositivo(s) afectados

- Contenedor LXC en nodo Proxmox Apollo  
- VLAN 20 (DMZ)  
- Servicios internos accesibles vía proxy

---

## Plan de acción

1. Completar previamente **RFC-2025-0008-NET** para creación/configuración de VLAN 20.  
2. Crear contenedor LXC en VLAN 20 con IP fija (ej. 10.0.20.10).  
3. Instalar Docker y Docker Compose.  
4. Desplegar Nginx Proxy Manager usando docker-compose con configuración básica.  
5. Configurar firewall para permitir acceso HTTP/HTTPS a NPM.  
6. Acceder a la interfaz web y configurar usuario administrador.  
7. Configurar certificados TLS (Let's Encrypt).  
8. Configurar reglas de proxy inverso para servicios existentes.  
9. Documentar configuración y credenciales.

---

## Riesgos

- Problemas de seguridad si la interfaz no se configura adecuadamente.  
- Fallos en la renovación automática de certificados TLS.

---

## Mitigación

- Verificar puertos disponibles antes de la instalación.  
- Restringir acceso a la interfaz de administración por IP o VPN.  
- Monitorizar logs y estado de certificados.

---

## Rollback

- Detener y eliminar el contenedor LXC de NPM en caso de fallo.  
- Restablecer accesos directos temporales a servicios sin proxy.  
- Revisar configuración y reintentar despliegue.

---

## Dependencias

- Depende de la finalización exitosa de **RFC-2025-0008-NET** (Creación de la VLAN 20 – DMZ).

---

## Resultado

Instalación completada con éxito. NPM está operativo y gestionando el proxy inverso según lo planificado, sin incidencias durante la implementación.
