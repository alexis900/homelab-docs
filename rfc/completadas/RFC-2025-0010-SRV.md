# RFC-ID:

RFC-2025-0010-SRV

# Título:

Exposición de Home Assistant a Internet vía Proxy inverso (Nginx Proxy
Manager)

# Tipo de cambio:

SRV / SEC / NET / IOT

# Fecha propuesta:

2025-08-02

# Estado:

Completado

# Criticidad:

Alta

# Justificación de criticidad:

Exponer Home Assistant a Internet implica riesgos de seguridad elevados,
pudiendo ser un vector de ataque si no se configura correctamente.
Además, afecta la disponibilidad y el acceso remoto a un servicio
crítico del hogar.

# Resumen:

Configurar Home Assistant para ser accesible desde Internet a través de
Nginx Proxy Manager (NPM) ubicado en VLAN 20, utilizando certificados
TLS válidos y aplicando medidas de seguridad para minimizar riesgos.

# Motivación:

Permitir acceso remoto seguro a Home Assistant para control y
monitorización desde fuera de la red local, facilitando integraciones
con servicios externos (por ejemplo, Google Home) y control remoto.

# Dispositivo(s) afectados:

-   Servidor Home Assistant (HA)

-   Nginx Proxy Manager (NPM) en VLAN 20

-   Router/Firewall (OPNsense) para reglas de NAT y firewall

-   Red VLAN 20 y VLAN 30

# Plan de acción:

-   Verificar que NPM está instalado y funcionando correctamente en VLAN
    20.

-   Configurar un host proxy en NPM apuntando a la IP y puerto de Home
    Assistant dentro de la red local.

-   Solicitar y configurar certificados TLS válidos para el dominio (por
    ejemplo, ha-amartinper.duckdns.org) en NPM.

-   Configurar reglas de firewall y NAT en OPNsense para permitir el
    tráfico HTTPS entrante hacia NPM en VLAN 50.

-   Limitar el acceso a la interfaz de NPM y HA con autenticación fuerte
    y, si es posible, con restricciones de IP o VPN para administración.

-   Probar acceso remoto a Home Assistant vía HTTPS desde Internet.

-   Monitorizar logs y tráfico para detectar accesos sospechosos.

-   Documentar configuración, certificados y credenciales.

# Riesgos:

-   Exposición a ataques externos (brute force, exploits).

-   Mal configuración de certificados TLS o proxy que cause
    inaccesibilidad.

-   Posibles problemas de rendimiento o caídas si el proxy falla.

-   Impacto en la seguridad general de la red doméstica.

# Mitigación:

-   Usar certificados TLS actualizados y firmados por CA confiable.

-   Configurar autenticación fuerte en Home Assistant y NPM.

-   Limitar puertos abiertos en firewall solo a los necesarios.

-   Mantener actualizados todos los componentes.

-   Implementar monitoreo continuo y alertas.

# Rollback:

-   En caso de problemas graves, revertir las reglas NAT y firewall para
    bloquear acceso externo a NPM.

-   Deshabilitar el host proxy para Home Assistant en NPM.

-   Restablecer accesos solo desde red local.

-   Revisar logs y corregir configuración antes de intentar nuevamente.

# Dependencias:

-   Instalación y correcto funcionamiento de Nginx Proxy Manager
    (RFC-2025-0009-SRV).

-   Configuración de VLAN 20 (RFC-2025-0008-NET).

-   Certificados TLS válidos (gestión en NPM).

# Resultado:

Completado el proceso de exposición de Home Assistant a Internet
mediante proxy inverso con Nginx Proxy Manager (NPM). La configuración
se realizó correctamente y el acceso externo funciona según lo esperado,
sin incidencias detectadas.
