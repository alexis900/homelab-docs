# RFC-ID:

RFC-2025-0017-SRV

# Título:

Actualización de Uptime Kuma de la versión 1.x a la versión 2.x

# Tipo de cambio:

SRV

# Fecha propuesta:

2025-11-02

# Estado:

Completada

# Criticidad:

Media

# Justificación de criticidad:

El cambio afecta a un servicio de monitorización central, aunque sin
interrumpir servicios productivos. La criticidad se considera media
debido al posible impacto temporal en la supervisión y a la necesidad de
garantizar compatibilidad con Node.js y dependencias del sistema.

# Resumen:

Actualización de la instancia actual de Uptime Kuma (instalada mediante
npm) desde la versión 1.x a la nueva versión 2.x, incorporando las
mejoras de seguridad, rendimiento y compatibilidad.

# Motivación:

Uptime Kuma v2 incluye una reestructuración interna basada en Node.js
20, optimización del uso de memoria, mejoras en autenticación y en la
interfaz, y soporte extendido a integraciones externas. La versión 1.x
quedará obsoleta y sin soporte activo a medio plazo.

# Dispositivo(s) afectados:

-   Servidor uptimekuma (instancia de Uptime Kuma)

# Plan de acción:

-   Notificar la ventana de mantenimiento al equipo de sistemas.

-   Verificar la versión actual de Node.js (mínimo v18, recomendado
    v20).\
    Realizar copia de seguridad del CT.

-   Verificar el espacio libre en disco y permisos.

-   Actualizar dependencias y la propia aplicación:

    -   cd /opt/uptime-kuma

    -   git pull

    -   npm install \--production

-   Ejecutar las migraciones automáticas (si las hubiera).

-   Reiniciar el CT para poder acceder correctamente.

-   Comprobar acceso web, estado de los monitores y logs de aplicación.

-   Documentar la versión instalada y resultado del cambio.

# Riesgos:

-   Fallo en dependencias Node.js o incompatibilidad con la versión de
    npm.

-   Corrupción de la base de datos en caso de migración interrumpida.

-   Interrupción temporal de la monitorización durante el proceso.

# Mitigación:

-   Copia de seguridad y snapshot del CT para mayor seguridad.

-   Posibilidad de revertir a la versión anterior mediante restauración
    de backup

# **Resultado**:

Actualización completada satisfactoriamente.

El servicio se actualizó correctamente a **Uptime Kuma v2.0.1**,
conservando la configuración, monitores y alertas.

No se detectaron errores en logs ni incidencias tras la migración.

Se verificó conectividad con todos los endpoints y rendimiento estable.
