# RFC — Reducción de memoria RAM asignada a VM OPNsense

## Resumen

Reducción de la memoria RAM asignada a la VM OPNsense de 3,5 GB a 2 GB para optimizar el uso de recursos del host Proxmox.

---

## Motivación

- Optimizar el uso de recursos en el host Proxmox.  
- La VM OPNsense tiene un consumo real de 500–600 MB, la asignación previa era excesiva.  
- Liberar memoria para otras VMs sin impactar en el servicio de red.

---

## Justificación de criticidad

Cambio no disruptivo, reversible y sin impacto previsto en la disponibilidad del servicio. La VM OPNsense no usa servicios de inspección profunda (IDS/IPS) que requieran mayor RAM.

---

## Sistemas / Dispositivos afectados

- VM OPNsense  
- Host Proxmox (liberación de memoria)

---

## Plan de acción

1. Modificar la asignación de memoria RAM de 3,5 GB a 2 GB en Proxmox.  
2. Reiniciar la VM.  
3. Verificar correcto funcionamiento de firewall, NAT, DNS y VPN.  
4. Monitorizar uso de memoria y swap durante el funcionamiento normal.

---

## Riesgos

- Posible presión de memoria en caso de picos de carga inesperados.

---

## Mitigación

- Monitorización posterior del uso de RAM y swap.  
- Posibilidad de rollback inmediato aumentando de nuevo la memoria asignada.

---

## Plan de rollback

1. Aumentar memoria RAM de la VM nuevamente a 3,5 GB.  
2. Reiniciar la VM si fuera necesario.  
3. Verificar conectividad, firewall, NAT, DNS y VPN.  
4. Revisar logs y uso de recursos para asegurar estabilidad.

---

## Validaciones post-cambio

- [x] Firewall, NAT, DNS y VPN operativos  
- [x] Uso de memoria y swap dentro de límites normales  
- [x] No se detectaron errores ni incidencias  
- [x] Cambio validado y documentado

---

## Resultado

Consumo de memoria observado: ~461 MB.  
No se detectan errores, uso de swap ni impacto en el servicio. Cambio validado con éxito.

---

## Observaciones

- Mantener monitorización temporal para verificar posibles picos de memoria futuros.
