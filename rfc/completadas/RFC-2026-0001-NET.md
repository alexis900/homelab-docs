# RFC -- Actualización controlador Omada a versión 6.x

RFC-2026-0001-NET

**Título:** Actualización de Omada Controller de 5.15.24.19 a 6.0.0.25

**Tipo de cambio:** NET

**Fecha propuesta:** 2025-12-31

**Estado:** Completada

**Criticidad:** Media

# Justificación de criticidad:

El cambio implica migración de versión mayor del controlador de red,
afectando la gestión centralizada de switches, APs y gateways. Si hay
fallo, podría impactar la visibilidad y configuración de la red
corporativa temporalmente.

# Resumen:

Actualizar Omada Controller a la versión 6 para disponer de nuevas
funciones, mejoras de seguridad y compatibilidad con firmware reciente
de los dispositivos.

# Motivación:

-   Acceso a nuevas funcionalidades de la versión 6.

-   Mejoras de seguridad y estabilidad.

-   Preparación para futuras actualizaciones de dispositivos y firmware.

# Dispositivo(s) afectados:

-   Servidor Omada Controller

-   Switches gestionados

# 

# Plan de acción:

-   Realizar backup completo del controlador Omada versión 5.x.

-   Detener servicios del controlador 5.x.

-   Instalar la versión 6.0.0.25.

-   Verificar migración de base de datos y configuración.

-   Comprobar que todos los dispositivos se reconectan y aparecen en
    línea.

-   Revisar logs y alertas del controlador.

-   Ejecutar tests de conectividad y políticas de red.

-   Realizar backup post-actualización.

# Riesgos:

-   Dispositivos no migran correctamente.

-   Pérdida temporal de conectividad de gestión.

-   Configuraciones personalizadas pueden no aplicarse correctamente.

# Mitigación:

-   Backup completo previo a la actualización.

-   Plan de rollback a la versión 5.x si fuera necesario.

-   Verificación de logs y conectividad antes de validar cambio.

# Resultado:

La actualización de Omada Controller a la versión 6.0.0.25 se completó
exitosamente dentro del contenedor (CT). Se verificó que todos los
dispositivos gestionados (switches) aparecieran en línea y funcionando
correctamente. Todas las configuraciones, políticas y VLANs se migraron
correctamente y los dashboards reflejan los datos de tráfico esperados.

Se realizó un **snapshot previo** a la actualización y un **backup
completo** posterior, garantizando puntos de restauración seguros. No se
detectaron errores ni incidencias durante el proceso, y el servicio de
gestión de red continúa operativo sin interrupciones.
