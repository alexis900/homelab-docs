# RFC — Instalación de CT UniFi OS

## Resumen

Se propone desplegar un contenedor (CT) con **UniFi OS** en la infraestructura de virtualización existente para centralizar la gestión de dispositivos UniFi, incluyendo controladores de red, Wi-Fi y otros servicios asociados. El CT se configurará en **VLAN 99** para segmentar su tráfico de administración y garantizar aislamiento de la red de gestión.

---

## Motivación

El cambio se realiza para:

- Centralizar la gestión de la infraestructura UniFi.
- Facilitar actualizaciones y mantenimientos futuros sin afectar otros servicios.
- Mejorar seguridad mediante aislamiento en CT y uso de VLAN específica (VLAN 99).
- Establecer un entorno replicable y documentado para futuras expansiones.

---

## Justificación de criticidad

La criticidad se considera **media**, ya que un fallo impediría la gestión centralizada de la red UniFi, afectando controladores y dispositivos, pero no interrumpiría servicios esenciales de otros CTs ni la conectividad de clientes finales de manera directa.

---

## Sistemas / Dispositivos afectados

- CT destino para UniFi OS
- Host Proxmox
- Red interna donde se integrará UniFi OS (**VLAN 99**)
- Servicios dependientes del controlador UniFi (Wi-Fi, switches, seguridad de red)

---

## Plan de acción

Pasos detallados para ejecutar la instalación:

1. **Creación del CT de UniFi OS**  
   - Crear el CT con recursos apropiados y configurarlo en **VLAN 99**.  
   - Ajustar CPU, memoria y almacenamiento según necesidades reales.

2. **Arranque del CT**  
   - Iniciar el CT y verificar que se comunica correctamente dentro de **VLAN 99**.

3. **Instalación de UniFi OS**  
   - Actualizar el sistema base.  
   - Instalar UniFi OS siguiendo la documentación oficial.  
   - Configurar los servicios y puertos necesarios.

4. **Verificaciones**  
   - Confirmar que el CT responde por SSH y web UI.  
   - Validar que la red del CT se integra correctamente en **VLAN 99** y la infraestructura existente.

5. **Tests funcionales**  
   - Iniciar sesión en UniFi OS.  
   - Verificar detección y gestión de dispositivos UniFi existentes.

6. **Backup post-cambio**  
   - Crear snapshot del CT instalado y operativo.  
   - Exportar configuración de UniFi OS para restauración rápida si es necesario.

---

## Riesgos

- CT no arranca correctamente por configuración de red o recursos insuficientes.
- Conflictos de puertos con otros servicios.
- Fallo en la migración de configuraciones previas de UniFi.

---

## Mitigación

- Comprobar compatibilidad de recursos antes de la instalación.
- Reservar puertos dedicados para UniFi OS.
- Mantener snapshot previo para rollback rápido.

---

## Plan de rollback

1. Detener CT de UniFi OS.
2. Restaurar snapshot del host o CT previo a la instalación.
3. Verificar funcionamiento de servicios existentes.
4. Validar conectividad de red y gestión de dispositivos UniFi antiguos si fuera necesario.

---

## Validaciones post-cambio

- [X] CT operativo y accesible por SSH y web UI.
- [X] Configuración de red correcta en **VLAN 99**.
- [X] Puertos de servicios UniFi activos y sin conflictos.
- [X] Logs sin errores críticos.
- [X] Snapshot y backup de configuración realizados.

---

## Resultado

La instalación del CT de UniFi OS se completó satisfactoriamente. Todos los servicios están operativos y la VLAN 99 funciona correctamente para la gestión del sistema.

---

## Observaciones

- Se recomienda monitorizar la primera semana el uso de CPU/memoria del CT y la estabilidad de la red UniFi.  
- Documentar versión exacta de UniFi OS instalada para futuras actualizaciones.
