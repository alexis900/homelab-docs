# RFC — Migración de UniFi Network Server a UniFi OS Server

## Resumen

Migrar el controlador **UniFi Network Server** autoalojado al nuevo **UniFi OS Server** para alinearnos con el estándar actual de Ubiquiti y disponer de funcionalidades completas de UniFi OS.

## Motivación

- UniFi OS Server es el estándar actual para self‑hosting y reemplaza al UniFi Network Server legacy.
- Mantener la plataforma soportada reduce riesgo operativo y facilita futuras actualizaciones.
- Compatibilidad con funcionalidades modernas (Site Manager/Organizaciones/IdP).

## Alcance / Sistemas afectados

- Controlador UniFi Network autoalojado (actualmente en Proxmox).
- APs UniFi U7 Lite y dispositivos gestionados por el controlador.
- Administración remota (Site Manager) si está habilitada.
- No aplica a OPNsense ni a otros servicios fuera del controlador.

## Plan de acción

1. Verificar versión actual, host y dependencias del controlador UniFi Network Server.
2. Generar backup completo en **Settings → Control Plane → Backups** y exportar site si aplica.
3. Preparar **CT nuevo** para UniFi OS Server (VLAN/CPU/RAM/storage y snapshot inicial).
4. Programar ventana de mantenimiento y comunicar posible indisponibilidad del controlador.
5. Detener **UniFi Network Server** en el CT actual (no desadoptar dispositivos).
6. Instalar **UniFi OS Server** en el CT nuevo según SO del host.
7. Migración:
   - Importar mediante **Site Export** o restaurar backup del sitio.
8. Verificar adopción de dispositivos, SSIDs, VLANs y políticas.
9. Revisar cambios de puertos/servicios (captive portal pasa a **8444** desde **8843**) y ajustar reglas si aplica.
10. Rehabilitar acceso remoto (Site Manager) y validar acceso admin.
11. Dejar el CT legacy apagado pero disponible durante el periodo de observación.

> Incluir backup adicional tras la migración y actualizar la documentación de servicios.

## Riesgos

- Pérdida temporal de gestión del controlador durante la migración.
- Incompatibilidades en puertos/servicios (captive portal).
- Problemas de migración de configuración o adopción.

## Mitigación

- Backup completo y exportación previa del sitio.
- Ventana de mantenimiento con pruebas controladas.
- Mantener el CT legacy apagado pero disponible para rollback.

## Plan de rollback

1. Detener UniFi OS Server en el CT nuevo.
2. Arrancar el **CT legacy** con UniFi Network Server.
3. Restaurar backup del controlador y validar adopciones.

## Validaciones post-cambio

- [x] Controlador accesible por GUI y API
- [x] Dispositivos adoptados y online
- [x] SSIDs/VLANs operativos
- [x] Logs sin errores críticos
- [x] Monitorización sin alertas relevantes

## Cambios DNS (si aplica)

- No se esperan cambios DNS.

## Resultado (completar al cerrar)

- Migración completada correctamente y sin incidencias relevantes.

## Observaciones

- El captive portal cambia de **8843** (legacy) a **8444** en UniFi OS Server.
