# RFC — Documentación de IPs secundarias de Proxmox en VLAN 1

## Resumen

Se documentan las IPs secundarias de los nodos Proxmox **Zeus** y **Hermes** en la red **VLAN 1**, manteniendo como referencia principal las IPs de gestión en **VLAN 99**. El objetivo es dejar claro que estas direcciones son de contingencia o uso secundario, no de administración principal.

## Motivación

- Evitar ambigüedad entre la IP de gestión real y una IP alternativa.
- Mantener la documentación alineada con el estado operativo actual.
- Facilitar tareas de soporte cuando se necesite una referencia en VLAN 1 sin mezclarla con el plano de gestión.

## Alcance / Sistemas afectados

- Nodo Proxmox **Zeus**.
- Nodo Proxmox **Hermes**.
- Documentación operativa en `Doc-Red/Dispositivos.md`, `Doc-Red/Cambios.md` y `Doc-Red/Doc-Red-full.md`.

## Plan de acción

1. Actualizar `Doc-Red/Dispositivos.md` para reflejar las IPs secundarias de Zeus y Hermes.
2. Regenerar `Doc-Red/Doc-Red-full.md` con `./scripts/build-doc-red.sh`.
3. Registrar el cambio en `Doc-Red/Cambios.md`.
4. Cerrar este RFC como completado una vez verificada la coherencia documental.

## Riesgos

- Que la redacción se interprete como si VLAN 1 fuese la red principal de gestión.
- Que la documentación agregada quede desincronizada respecto a la fuente operativa.

## Mitigación

- Mantener la nota explícita de que VLAN 99 sigue siendo la red de gestión.
- Regenerar el documento consolidado tras cada cambio en `Doc-Red/`.

## Plan de rollback

1. Revertir los cambios en `Doc-Red/Dispositivos.md` y `Doc-Red/Cambios.md`.
2. Regenerar `Doc-Red/Doc-Red-full.md`.
3. Volver a la redacción anterior si se decide eliminar la referencia secundaria.

## Validaciones post-cambio

- [x] Zeus muestra `10.0.1.125/24` como IP secundaria.
- [x] Hermes muestra `10.0.1.230/24` como IP secundaria.
- [x] La nota deja claro que la gestión principal sigue en VLAN 99.
- [x] `Doc-Red/Doc-Red-full.md` está regenerado.

## Cambios DNS (si aplica)

- No aplica.

## Resultado (completar al cerrar)

- La documentación de Proxmox distingue correctamente entre IP de gestión e IP secundaria.
- La referencia quedó alineada en la documentación principal y en el historial de cambios.

## Observaciones

- Si en el futuro se retiran estas IPs secundarias, conviene retirar también la referencia documental para evitar ruido.
