# RFC — Migración de DHCP normal a Kea en OPNsense 26.1

## Resumen

Se realizará la migración de la gestión del DHCP para todas las VLANs normales (no PXE) desde ISC DHCP legacy a Kea DHCP en OPNsense 26.1.  
El objetivo es preparar la infraestructura para futuras versiones de OPNsense, mejorar el soporte de DHCP moderno y mantener ISC DHCP solo para la VLAN PXE legacy.

---

## Motivación

- Sustituir ISC DHCP legacy, que ya no recibe soporte, por Kea DHCP para DHCP normal.  
- Garantizar compatibilidad futura con OPNsense 26.x y 27.x.  
- Mantener PXE funcional mediante ISC plugin legacy, evitando interrupciones.  
- Simplificar la gestión de reservas, rangos y opciones DHCP.

---

## Justificación de criticidad

El cambio afecta directamente a la asignación de IP en toda la red homelab. Un fallo podría impedir que los clientes obtengan IP, perder conectividad o afectar servicios dependientes de DHCP.  
Se considera criticidad **media** porque la VLAN PXE se mantiene separada y el riesgo se puede mitigar con rollback.

---

## Sistemas / Dispositivos afectados

- OPNsense 26.1  
- VLANs internas no PXE  
- Clientes DHCP en LAN y VLANs secundarias  
- Servidores y servicios dependientes de DHCP

---

## Plan de acción

1. **Backup completo** de configuración OPNsense y snapshot.  
2. Verificar estado actual de ISC DHCP en VLANs no PXE.  
3. Activar Kea DHCP en OPNsense para VLANs normales:  
   - Configurar rangos IP  
   - Configurar reservas MAC  
   - Configurar opciones DHCP (DNS, gateway)  
4. Desactivar ISC DHCP en las VLANs normales, manteniendo activo solo para PXE.  
5. Validar que los clientes obtienen IP desde Kea DHCP.  
6. Verificar logs, conectividad y servicios dependientes de DHCP.  
7. Documentar configuración final y cambios realizados.

---

## Riesgos

- Clientes que no obtienen IP tras desactivar ISC DHCP.  
- Conflictos de rango entre ISC y Kea si ambos están activos simultáneamente.  
- PXE podría verse afectado si ISC DHCP se desactiva en la VLAN equivocada.  

---

## Mitigación

- Activar Kea **antes** de desactivar ISC DHCP en cada VLAN.  
- Mantener VLAN PXE con ISC activo durante la migración.  
- Realizar pruebas de obtención de IP en clientes antes de desactivar ISC.  
- Backup y snapshot completos para rollback rápido.

---

## Plan de rollback

1. Reactivar ISC DHCP en las VLANs afectadas si Kea falla.  
2. Validar que los clientes recuperan IP correctamente.  
3. Restaurar configuración desde snapshot si el problema persiste.  
4. Revisar conectividad y servicios dependientes de DHCP.

---

## Validaciones post-cambio

- [ ] Clientes obtienen IP correctamente desde Kea DHCP.  
- [ ] VLAN PXE sigue funcional con ISC plugin legacy.  
- [ ] Servicios dependientes de DHCP accesibles.  
- [ ] Logs sin errores críticos.  
- [ ] Backup actualizado con la nueva configuración.

---

## Observaciones

- Documentar cualquier incidencia durante la migración para referencia futura.  
- Revisar opciones avanzadas de Kea si se requiere soporte de DHCPv6, prefix delegation o reservas complejas en el futuro.
